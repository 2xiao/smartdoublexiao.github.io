Hexo实现个人博客搭建部署美化

周末搭建了自己的博客，我使用的是GitHubPages + Hexo+ Even，一种简单高效的实现方式。当然还有其他的实现方式，如：WordPress、GitHubPages + Jekell等等。

本地环境准备

关于创建Github仓库，Git安装，Nodejs安装我就不再赘述。

Hexo安装

Hexo是一个极速、简单且强大的静态博客架构，安装和使用十分便捷：

打开终端，输入Hexo的安装命令：
​
npm install hexo-cli -g

定位到你希望存储博客的目录下:
​
cd *

初始化本地博客仓库：
​
hexo init 你的用户名.github.io // 尽量和Github仓库使用同一个名称

基本的博客框架就已经搭建完了，Hexo安装完成后默认使用landscape主题，终端定位到xxx.github.io文件夹下，输入下面的命令预览：
​
hexo s

在浏览器上输入网址：http://localhost:4000/ ,Press Ctrl+C to stop. 就能预览到landscape主题下的博客效果。

主题安装

我原本用的是大家都在用的NexT主题，但是看到@k.Chen安利@ahonn制作的简约主题Even，确实蛮好看的，所以就搬过来用了，步骤如下：

将终端定位到xxx.github.io目录下，下载并安装Even主题：
​
cd xxx.github.io

git clone https://github.com/iissnan/hexo-theme-even 

cd even/

bin/install

在文件目录中打开_config.yml(./xxx.github.io/config.yml)启用Even主题：
​ 

# Extensions

## Plugins: http://hexo.io/plugins/

## Themes: http://hexo.io/themes/

theme: even

Even主题就安装好了，其他诸如更改名字、添加社交信息、增加自定义页面、更改LOGO，都在xxx.github.io/themes/even/doc/doc_zh.md中，不再一一赘述。

博客部署

终端输入下面的命令完成安装自动部署发布工具：
​
sudo npm install hexo-deployer-git –save

在文件目录中打开_config.yml(./xxx.github.io/config.yml),编辑最后的 Deployment 和 deploy 配置项：
​ 

# Deployment

## Docs: https://hexo.io/docs/deployment.html

deploy:

 type: git

​
​
deploy:

type: git

repo: <repository url>

branch: [branch]

message: [message]

在本地确认博客效果后，就可以将md文件生成静态网页上传至GithubPages，在终端定位到xxx.github.io目录下，执行下面的命令即可：
​
hexo clean #清楚网页缓存

hexo g #生成静态网页

hexo d #开始部署

//当然也可以使用一次性命令：

hexo clean && hexo g && hexo d

首次部署时，终端会让你输入Github的用户名和密码，建议使用 SSH 加密的仓库地址（参看Github 官方文档配置 SSH 免密操作），现在，所有人都可以通过 http://.github.io 来访问你的博客了。

绑定域名

现在我们已经可以无碍的使用 Hexo 来写博客并部署到GitHub上了，可是有一个地方不太优雅，那就是域名。因为我们使用的是 Github 提供的二级域名 github.io，一来太长，二来也不太够自定义、个性化。

所以，我们可以去买一个域名，然后解析到 Github 去，国内的万网、西部数码，国外的 Godady 等网站都可以购买。我的域名 .tech 每年才 5 元，价格实惠。实在不济，还有 .tk，.ml，.cf，.ga 四个免费域名提供开放注册（为互联网的免费精神干杯🍻）。

购买域名之后，我们只需要在购买域名的网站把 DNS 解析创建一个CNAME记录和两个 A 记录到：
​
@ A 192.30.252.153

@ A 192.30.252.154

WWW CNAME xxx.github.io

之后在自己的博客根目录下创建一个 CNAME 文件，里面放你解析的域名。

之后修改根目录下的 _config.yml 文件，把站点地址更新成新的绑定的域名即可
​ 

# URL

## If your site is put in a subdirectory, set url as 'http://yoursite.com/child' and root as '/child/'

url: http://yoursite.com

现在，就可以使用自己的域名访问自己的博客了，例如： http://xxwu.tech

评论CSS美化

为了更好地交流互动，我使用了多说的评论模块。首先，在多说网站登录，点击我要安装，进入后台管理，在设置中输入自己的域名：xxx.duoshuo.com，其中xxx就是你的duoshuo_shortname

然后需要自己创建Board界面,打开终端，定位到xxx.github.io目录下,执行下面的命令:
​
hexo new page board

创建完成后，在对应的目录下找到index.md文件，进行如下的修改：

​ 

title: board

date: 2016-11-30 17:16:58

layout: page

---

**最近访客**

<div class="ds-recent-visitors" data-num-items="28" data-avatar-size="42" id="ds-recent-visitors"></div>

打开xxx.github.io/themes/even/_config.yml,修改Menu模块代码：
​
menu:

Archives: /archives/

Categories: /categories/

Tags: /tags/

About: /about/

Board: /board/

修改comments模块，其中post是指各篇文章和留言板的评论功能是否启用，而about是指关于页面的。
​ 

# comments

comments:

 post: true

 about: false

以及后面的：
​ 

 # Comments

## Duoshuo

duoshuo_shortname: xxx(上文提到的多说短域名)

## Disqus

#disqus_shortname: Your disqus ID
