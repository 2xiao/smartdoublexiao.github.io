如何预览GitHub里的Demo？

早上逛师姐的博客，发现她做过的demo都放在自己的域名分支下，我一开始以为是将项目代码copy到个人博客仓库里去，然后就像博客里的一篇文章一样打开，结果出来的效果是：会自动加上 Header、Footer 和Comment 等模块，我还特别傻的去theme的配置文件里去改了好久，后来明白只需要在github上建立 gh-pages 分支就可以预览了。

言归正传，预览Github里的网页或Demo的方法有两种。

方法一：链接加前缀

第一种是直接在项目源代码页面链接前缀那加上 http://htmlpreview.github.com/? , 如:

http://htmlpreview.github.com/?https://github.com/SmartDoubleXiao/ife/blob/master/6.html

适合写完代码预览效果，但是比较麻烦，而且强迫症宝宝受不了。

方法二：github建分支

第二种是在Github上建立 gh-pages 分支。

为什么要建立gh-pages分支呢，因为github项目的静态页面解析需要这个名字的分支。

进入到你想要上传的项目文件夹下：

cd text

git初始化

git init

创建gh-pages分支

git checkout --orphan gh-pages

添加文件到暂存区

git add .

添加信息

git commit -m "This is add message"

或者不写上面的git add .直接写git commit -a -m \"First pages commit\"这个-a参数我查了之后说是对git add .的替代，但我不建议大家使用。

添加仓库

git remote add origin git@github.com:username/project.git

部署你的项目到github

git push origin gh-pages

现在你的github profile 中的相应项目下就有一个gh-pages了，直接访问 username.github.io/项目/网页.html就可以直接预览demo,例如：http://SmartDoubleXiao.github.io/ife/7.html . 如果你将github仓库解析到了自己的域名下，就可以直接输入域名/项目/网页.html访问了，例如：xxwu.tech/ife/7.html.
